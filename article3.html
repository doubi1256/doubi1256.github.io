5分钟入门/execute指令

/execute的简单使用方法 

首先我们进入游戏，输入/execute加空格，然后就能看到很多的子命令

因为/execute真的每个标签都搞懂很难，我这里只会讲最基础的

run 

首先是最简单的run（执行）

 基本格式： /execure run 命令（命令不带/) 

示例：

 /execute run kill @s（执行后会自我了断） 

其效果和/kill @s相同。

 有同学就会问了，那逗比，这个execute指令没啥用的样子啊我为什么要学！ 

别急，execute指令必须要和其他子命令一起运用才能玩出花哦 

as

 这个标签可重要了，会改变命令的执行者 如果你有选择器基础的话就很好讲了，不过没有也没关系，直接上示例 

示例： /execute as @e[type=villager] run kill @s （全部的村民自我了断）

 这里的@e后面的[type=]就是选择器了。[type=]选择了实体，[name=xx]就是选择名字为xx的实体。 

我使用了as子命令，就改变了指令的效果。

at 

这个标签也很重要，会改变命令的执行位置 这么说听起来很抽象，同样我会使用范例来给大家讲解 

首先我们假设，世界内存在6只村民

 示例：

 /execute at @e[type=villager] run summon cat ~ ~ ~ (在所有村民的脚下生成一只猫) 

如果我将at换成as，结果则是

 示例：

 /execute as @e[type=villager] run summon cat ~ ~ ~ (在我的脚底下生成6只猫) 

这很简单，as只改变了命令的使用者，而没有改变命令的使用位置 

而at改变了命令的使用位置，将相对坐标的坐标轴定义到了所选对象上。

in 

或许有人会和at搞混，但这个子命令的作用是改变命令执行的维度 

这个子命令很常见，比如我们按下F3+C，就可以复制自身的坐标 

示例：

 /execute in the_end run tp 114 51 419 (传送到末地的坐标为114 51 419的地点) 

下面是简单的条件命令部分，我只讲if/unless里的block

if/unless(如果/除非) 

请用字面意义来理解。 

if是如果...就执行命令 而unless是执行命令，除非... block是其中的的一个参数

 格式为 if/unless block 坐标 方块id 

接下来让我们把所有元素结合起来—— 

示例：

 /execute as @e[type=villager] at @s if block ~ ~-1 ~ white_wool run kill @s 

(如果村民站在白色羊毛上，就会自我了断) (这里必须指定命令执行位置，否则一只村民站在白色羊毛上，所有村民都会自我了断) 

（下面unless同理，如果村民没站在白色羊毛上，就会自我了断）

 /execute as @e[type=villager] at @s unless block ~ ~-1 ~ white_wool run kill @s
